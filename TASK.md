# Промпт: Система мультиаккаунт-автоматизации для YouTube на Node.js

## Контекст задачи
Разработать инструмент на Node.js для управления 10+ аккаунтами YouTube с полной изоляцией сессий. Каждый аккаунт работает через выделенный прокси с подменой WebGL-параметров. Система собирает OAuth-токены и автоматически редактирует метаданные видео (названия, описания, субтитры) на разных языках.

## Технические требования

### 1. Архитектура (Двухфазная система)

**Фаза 1: Сбор токенов (Onboarding)**
- Локальный сервер на Express.js
- Фронтенд-панель управления на чистом HTML/CSS/JS
- Запрос на авторизацию должен идти через Proxy (будет несколько прокси для нескольких аккаутов)
- Подмена WebGL через stealth-плагин или кастомное расширение
- Ручное прохождение OAuth 2.0
- Сохранение refresh_token в БД (какой-то локальный файл используй для бд)

**Фаза 2: Автоматизация (API-взаимодействие)**
- HTTP-клиент с поддержкой прокси через tunnel

### 2. Компоненты системы

#### А. Фронтенд (Панель управления)
- Список аккаунтов с карточками
- Выбор прокси для каждого аккаунта (из предопределенного пула)
- Поле для ввода email аккаунта
- Кнопка "Авторизовать" → открывает окно с Puppeteer
- Статус-индикаторы (⚪ не авторизован, ✅ авторизован)
- Обработчик postMessage для получения статуса от всплывающих окон

#### Б. Текущий код
Текущий код уже работает с одим аккаунтом и позволяет менять заголовок, описение, субтитры видео для нескольких языков, этот код нужно сохранить и переиспользовать. 

#### В. Прокси
Будет использовать HTTPS-прокси. Данные для проски смогу добавить в конце.

#### Г. WebGL
Если WebGL не проверяется, то его можно не подменять.

#### Д. OAuth
Будет несколько гугл аккаунтов (10 штук). Каждый этот пользователь самостоятельно зайдёт в Google platform и создать там oauth пользователя (даже в тестовом режиме). Скачает client_secret.json и добави в каку-то папку в проекте. И каждого пользователя нужно будет отдельно авторизовать в системе.
